# precisionmachining.co blog build

This repository uses a build-time MDX pipeline for the blog while keeping the site fully static HTML on Vercel.

## Add a new article
1. Create `content/articles/<slug>.mdx`.
2. Use this frontmatter schema:

```yaml
---
title: "CNC Milling: 3-Axis, 4-Axis, and 5-Axis Explained"
slug: "cnc-milling-guide"
date: "2025-02-05"
preview: "More axes means more capability — but also more cost..."
description: "Meta description text"
h1: "CNC Milling: 3-Axis, 4-Axis, and 5-Axis Explained"
category: "Processes"
icon: "default"
---
```

3. Add your markdown/MDX body below frontmatter.
4. Commit and push. Vercel runs `npm run build` and regenerates blog pages.

## Valid categories
- `Processes`
- `Materials`
- `Quality`
- `Sourcing`
- `Design`

## Local build
```bash
npm install
npm run build
```

## Generated files
Do **not** directly edit:
- `blog/<slug>/index.html`
- `blog/articles.json`
- `blog/index.html`

These files are generated by `scripts/build-blog.js`.
